Pasos de Migracion

1

Scan forense completo

VAS Home lupa - verificar tipo=E, listar tablas ODBC detectadas, revisar ICM esperado 4-8.

2

Implementar bridge REST (PREREQUISITO)

Levantar servicio REST en servidor con acceso a Sybase PROMETEO_II. Exponer endpoints HTTPS que GAS pueda llamar via UrlFetchApp.fetch(). Registrar URL base en drive_config.json.

3

Deconstruir el Excel

python src/engine/deconstructor.py archivo.xlsm - extrae modulos VBA y CSVs.

4

Convertir VBA a GAS

python src/engine/vba_to_gas.py package --drive-config drive_config.json

5

Ajustar GAS manualmente

Completar consultas SQL que el convertidor no pudo traducir. Probar cada funcion de lectura contra el bridge.

6

Deploy via clasp

npx @google/clasp create --type sheets luego npx @google/clasp push --force

7

Pruebas de integracion

Ejecutar cada macro que consulta Sybase. Comparar resultados contra Excel original.

8

Documentar

Crear MIGRACION_COMPLETA_id.md, registrar IDs en HITOS.md.

Checklist QA

  • Bridge REST responde en menos de 2s por consulta
  • Todas las tablas ODBC detectadas tienen endpoint en el bridge
  • GAS no tiene strings de conexion hardcodeadas
  • OAuth tiene scopes: spreadsheets, drive, script.external_request
  • Datos coinciden con Excel original
  • Manejo de errores para timeout de red
  • Menu BCI Migracion funciona correctamente
  • Log de ejecucion generado en Sheet
